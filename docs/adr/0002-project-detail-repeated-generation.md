# 0002. プロジェクト詳細画面の繰り返しバナー生成UI

**日付**: 2025-03-02

## 状況

プロジェクト詳細画面が「1回だけ生成」に見える表現になっており、同じプロジェクトで何度もバナーを生成・購入するユースケースに合っていなかった。LP 分析コンテキストを活かして繰り返し生成できるようにし、履歴をチャット風に確認・購入できるようにしたい。

## 決定

- **データ構造**
  - `GenerationRound`: `{ id, referenceImageUrl?, banners: GeneratedBanner[3] }`。1ラウンド = 参考バナー1枚（任意）+ 生成3パターン。
  - 履歴は `generationsAtom = atom<GenerationRound[]>` で保持。**末尾に push** するため、画面上は上（index 0）が古い・下が最新のチャット型になる。

- **UI**
  - ヘッダーから「3パターン生成完了」「再生成」を削除し、1回限り感をなくす。
  - 見出しを「生成されたバナー 3パターン」から「生成履歴」に変更。複数ラウンドを縦に並べて表示（各ラウンドは横3列）。
  - **固定フッター**（プロジェクト画面のみ）に「参考バナーをアップロードして生成」ボタンを配置。クリックでモーダルを開く。
  - モーダル: 参考バナー画像を1枚選択（任意）、「3パターン生成」で新ラウンドを追加。現状はモックで即時3パターン反映。

- **購入**
  - 購入完了時は `markBannerPurchased(setGenerations, bannerId)` で該当ラウンド内のバナーを `isPurchased: true` に更新。`PurchaseModal` は `generationsAtom` を参照する形に変更。

- **初期状態**
  - `generationsAtom` の初期値は 1 ラウンド（参考バナーなし・モック3枚）。既存の `MOCK_BANNERS` をテンプレート化し、`createMockBannersForRound(roundId)` で新ラウンド用の3枚を生成。

## 理由

- 固定フッターにしたのは「いつでもアクセスできる」という要件を満たすため。
- 履歴を配列の末尾に追加し、表示は先頭からにしたので、スクロールで上に遡る＝過去、下＝最新のチャット型になる。
- 参考バナーは任意にし、アップロードなしでも「3パターン生成」で新ラウンドを追加できるようにした（モックのため即時反映）。

## 結果

- プロジェクト詳細で複数回「参考バナーをアップロードして生成」から新ラウンドを追加できる。
- 生成履歴がラウンドごとに縦に並び、各ラウンドは横3列。購入は従来どおりモーダルで行い、履歴内の該当バナーが購入済みになる。
- 永続化・API は今回スコープ外。将来は GenerationRound を API で取得・保存する拡張を想定。
