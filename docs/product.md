# 前提
私はLLMを組み込んだプロダクト開発を学びたいマーケターです。
今まで数万枚の広告を考案→デザインしてきた知見を元に、超絶効果が良さそうなバナーを超絶簡単に生成できる感動体験を作りたいと思っています。

# プロダクト
AI広告バナー生成Webアプリ（静止画のみ、1080x1080）
ユーザーはLP情報（URL or テキスト）と参考バナー画像を渡す。
システムは3パターンの広告バナーを「プレビュー」として返す（透かし付き）。
気に入った1枚を¥500で購入すると、同じ画像の「透かし無し・高画質」をダウンロードできる（完全一致優先、再生成はしない）。

# プロダクト概要
## ターゲット
- スタートアップのインハウスマーケター
### ターゲットのニーズ
- デザイナーへの発注費用や構成考案工数の削減
  - 毎週20~30本の広告を考えるのが負担
  - デザイン費用や工数を削減できれば利益が直接増える
## 提供価値
- 自社LPと参考広告バナーをアップロードするだけで、そのまま配信できる品質(訴求内容, デザイン)の広告バナーが生成される
  - ターゲットは普段から競合や類似ターゲット商材のバナーをリサーチしている
  - 良さそうに思ったバナーをアップロードするだけで高品質なバナーが生成される

## ユーザーストーリー
1. ユーザーがLPのURLを貼り付ける
2. AIがLPを分析し、ターゲットのデモグラやインサイトを推測する
3. ユーザーが必要に応じてその内容を修正する
4. AIがその内容をコンテキストに保存する
5. ユーザーが広告ライブラリなどで効果良さそうなバナーを見つける
6. ユーザーがバナーをアプリに貼り付ける
7. AIがバナーの訴求力(アイキャッチの工夫, 読ませる工夫, タップさせる工夫, CVRをあげる工夫)を分析し、効果に寄与しそうな部分を横展開したCTRもCVRも高そうなバナー画像を3パターン、ウォーターマークあり低画質で3枚出力する
8. ユーザーが、効果良さそうなものを¥500で購入すると、ウォーターマークが外れて高画質になったものをダウンロードできる

## 制約
- 個人開発なので大規模な先行投資は行えない
- 常に黒字であり続ける必要がある

## 料金・回数
- 未購入ユーザーは最大5回までプレビュー生成可能（1回で3枚）
- 何か購入するごとに +5回分（=15枚）のプレビュー枠を付与

# 技術選定（固定）
- フロント: React + TypeScript（ViteでOK）
- API: Cloudflare Workers（Hono等の軽量FWは使って良い）
- 画像保存: Cloudflare R2
- DB: Firestore（Firebase Authも使って良い）
- 決済: Stripe Checkout + Webhook（Workers側で受ける）
- 画像生成: OpenAI gpt-image-1 を使用（まずは quality=medium で1024x1024生成）
- 透かし: サーバ側で合成してpreview画像を作る（元画像は透かし無しで保存しておく）
- 納品: 透かし無しの元画像を1080にアップスケール（簡易でOK）して署名付きURLでDLさせる

# 重要な実装方針
- 将来モデル差し替えしやすいように ImageProvider インターフェースを作り、OpenAI実装を注入する設計にする。Geminiにするかも。
- 完全一致のため、preview生成時に「透かし無し原本」を必ず保存しておき、表示用に透かし付きを別保存する
- 同一入力（LP+参考画像）が同じ場合はキャッシュできるように、入力ハッシュを保存する
- 不正対策: Turnstile（または簡易レート制限）と、Firestoreでの回数管理を入れる（MVP水準でOK）
- エンドポイントはRESTでOK。ジョブは同期でも良いが、状態管理（pending/succeeded/failed）は実装する
- Agentモードで実装する場合でも、必ず詳細な実装方針を提案して、GOが出てから実装する。GOが出るまではコードを編集してはいけない

# 注意
- 画像の透かし合成は「サンプル実装」で良い（Canvas/wasm/ライブラリは現実的な選択を提案して採用して）
- OpenAIの呼び出しはSDKでもHTTPでもOK。APIキーは環境変数で管理。
- Cloudflare Workersで動かせる前提で、Node専用APIに依存しないように注意。
- 生成に失敗した場合のリトライはMVPでは1回までで良い。
